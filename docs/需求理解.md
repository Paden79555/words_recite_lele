## 2026-06-10 18:52

## 需求概述
面向深圳中考英语薄弱学生，网页需要从展示型 Demo 升级为可在微信内每天打开使用的背单词工具。核心价值不是“信息多”，而是让学生打开后立刻知道今天背什么、为什么背这些、背完有什么反馈；家长或管理员则能低成本看到完成率、薄弱词和风险趋势。

## 功能需求
### P0 - 必须实现
- [ ] 手机首屏直接进入今日背诵任务，弱化宣传内容和复杂仪表盘。
- [ ] 背诵流程遵循“先回忆 → 看答案 → 认识/忘了评分”。
- [ ] 任务队列优先展示到期复习词、错词和低掌握词。
- [ ] 每日完成进度、总掌握率、连续天数、待复习词数量必须清晰可见。
- [ ] 管理员看板展示总掌握、今日完成、高风险词、薄弱词清单和 7 天趋势。
- [ ] 词库从少量样例扩充为中考高频样例，并明确仍需导入完整官方词库。

### P1 - 重要功能
- [ ] 支持每日目标和任务策略设置。
- [ ] 支持读音、稍后复习、报告导出。
- [ ] 强化游戏化反馈，包括金币、闯关地图和记忆热力图。

### P2 - 可选功能
- [ ] 增加拼写测验、听写、例句选择题。
- [ ] 支持 CSV 导入完整词库。
- [ ] 支持跨设备同步和登录。

## 非功能需求
- 性能：纯静态页面，首屏无需外部接口，适合 GitHub Pages。
- 安全：学习数据仅保存在本机 localStorage，不上传个人数据。
- 可用性：主要适配 390px 左右手机竖屏，按钮尺寸适合拇指操作。
- 可维护性：词库独立为 words.js，便于后续替换完整词库。

## 边界与约束
- 不包含账号体系、云同步和多人班级管理。
- 不声称当前词库等于完整深圳中考官方词库。
- 仍以单人本机学习为主，清缓存或换设备会丢失数据。

## 验收标准
### 手机背诵主流程
- Given 学生用手机打开页面，When 点击“开始今日背诵”，Then 页面显示一个单词卡，并可完成看答案、认识、忘了、稍后流程。

### 进度反馈
- Given 学生完成一个单词评分，When 回到卡片，Then 今日进度、金币、闯关地图和看板数据同步更新。

### 管理员看板
- Given 家长打开看板，When 查看页面，Then 能在 10 秒内判断今日是否完成、薄弱词有哪些、整体掌握是否提升。

### 词库边界
- Given 用户查看设置或需求页，When 阅读上线提示，Then 能明确知道当前词库是高频样例而非完整官方词库。

## 2026-06-11 词库真实性纠偏

### 背景
用户指出“样例全量”仍只有 234 词，不应被理解为 2000 词完整词库。deep-research 工作流核验后，未发现可公开访问、可核验的深圳官方/半官方约 2000 词完整英语词汇表；教育部《义务教育英语课程标准（2022年版）》可作为课程方向依据，但不是可直接复制的深圳中考完整词表。

### 需求修正
- 页面必须明确区分“目标 2000 词”和“当前已导入词数”。
- 不再使用容易误导的“样例全量”命名，应改为“当前样例库 / 已导入样例”。
- 不主观伪造 2000 词，也不复制未授权第三方完整词表。
- 管理员需要看到词库完整性、来源可信度、导入格式和补全流程。
- 排位/巅峰赛可以按 2000 目标设计，但必须显示当前数据覆盖率。

### 验收标准
- 学生页和管理员页都能看到“已导入 X / 目标 2000”。
- 词库选择不再把 234 词称为完整全量。
- 管理员页提供 CSV/JS 导入说明和来源风险提示。
- README 明确当前非官方完整词库。
