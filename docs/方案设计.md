## 技术方案概述
保持 GitHub Pages 纯静态部署方式不变，将页面从“宣传 + 仪表盘”重构为“移动端背诵 App”。核心改动是移动优先的信息架构、独立词库文件、稳定的复习队列、非随机趋势数据和更清晰的管理员看板。

## 技术选型
| 类别 | 选择 | 理由 |
|------|------|------|
| 语言 | HTML/CSS/JavaScript | GitHub Pages 可直接托管，无构建门槛 |
| 框架 | 无框架 | 当前规模小，避免引入构建和依赖复杂度 |
| 数据存储 | localStorage | 单人本机使用足够，微信内打开也可保存 |
| 词库 | words.js 全局数组 | 便于后续替换为完整词库或导入生成 |

## 架构设计
- 表现层：index.html 负责四个底部 Tab：背诵、看板、设置、需求。
- 业务层：app.js 负责复习队列、评分、进度、图表渲染和交互。
- 数据层：words.js 提供词库，localStorage 保存学习状态。

## 模块划分
| 模块 | 职责 | 依赖 |
|------|------|------|
| 背诵页 | 今日任务、单词卡、评分、读音、稍后 | words.js、localStorage |
| 看板页 | 总掌握、完成率、薄弱词、趋势 | app.js 学习状态 |
| 设置页 | 每日目标、任务策略、报告导出、重置 | app.js 学习状态 |
| 词库 | 中考高频样例词数据 | 无 |

## 数据模型
### Word
- word：英文单词
- phonetic：音标
- meaning：中文释义
- example：例句
- level：分类标签

### WordProgress
- id：词库索引
- score：掌握度 0-100
- correct：正确次数
- wrong：错误次数
- intervalIndex：当前复习间隔索引
- nextReview：下次复习日期
- reviewedToday：今日是否已复习
- history：历史评分记录

## 关键设计决策
| 决策点 | 选择 | 理由 |
|--------|------|------|
| 首屏结构 | 今日任务优先 | 手机端用户打开后应立刻学习 |
| 趋势图 | 使用真实/种子 daily 数据 | 避免刷新随机变化导致看板不可信 |
| 词库文件 | 拆出 words.js | 后续替换完整词库更安全 |
| 可视化 | 进度圆环、闯关地图、热力柱状图 | 比纯文本更适合薄弱学生坚持 |
| 管理员页 | 指标摘要 + 薄弱词清单 | 家长只需要判断是否完成和哪里薄弱 |
